# API Route Plan

These routes are planned only. They should not be implemented in Phase 8.

All routes require schema review before implementation. CRUD should start with Paper only in Phase 10.

## Paper Routes

- `GET /api/papers`
- `GET /api/papers/:id`
- `POST /api/papers`
- `PATCH /api/papers/:id`
- `DELETE /api/papers/:id`

## Research Domain Routes

- `GET /api/research-domains`
- `GET /api/research-domains/:id`

## Dataset Routes

- `GET /api/datasets`
- `GET /api/datasets/:id`

## Model Routes

- `GET /api/model-results`
- `GET /api/model-results/:id`

## Weekly Progress Routes

- `GET /api/weekly-logs`
- `POST /api/weekly-logs`
- `PATCH /api/weekly-logs/:id`

## Decision Log Routes

- `GET /api/decisions`
- `POST /api/decisions`

## Implementation Notes

- These routes are planned only.
- They should not be implemented in Phase 8.
- They require schema review before implementation.
- CRUD should start with Paper only in Phase 10.
- Routes should remain aligned with `data_entities.md`.
- No live API routes should exist until backend implementation is explicitly approved.
