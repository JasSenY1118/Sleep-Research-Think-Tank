# Sleep Research Think Tank: Agent Rules

## Project Purpose

Sleep Research Think Tank is a clean academic website for current sleep research. The final product should organize research materials and progress across these modules:

- Dashboard
- Research Map
- Paper Library
- Dataset Viewer
- Model Results
- Weekly Progress

## Working Rules

- Work one phase at a time.
- Never add unrelated features.
- Never rewrite the whole app unless explicitly requested.
- Never create duplicate components if an existing component can be reused.
- Do not touch unrelated files.
- No new dependencies unless justified.
- Keep the design minimal, academic, and professional.
- Use a restrained palette: deep teal, muted navy, warm off-white, soft grey.
- After each run, report changed files, reason for changes, verification steps, and suggested checkpoint.
- Always check that the app still runs.
- Keep placeholder content clearly labelled.
- Do not invent real scientific papers, results, or statistics.

## Content Standards

- Label all draft or placeholder material as placeholder content.
- Use citations only when source details are available.
- Separate verified research facts from planning notes.
- Do not fabricate author names, publication titles, datasets, metrics, benchmark results, or statistical findings.

## Implementation Standards

- Prefer existing app structure, components, utilities, and styling conventions once the app exists.
- Keep changes narrowly scoped to the requested phase.
- Reuse components before creating new ones.
- Document meaningful decisions in `00_master_index/decision_log.md`.
- Track active and completed work in `00_master_index/task_tracker.md`.
- Update weekly progress in `00_master_index/weekly_log.md`.

## Reporting Template

After each implementation run, report:

1. Files changed.
2. Why each file changed.
3. Verification steps performed.
4. Whether the app still runs.
5. Suggested checkpoint name.
